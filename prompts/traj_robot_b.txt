## II. Efficiency and Time-Step Optimization
Task planning must explicitly consider execution efficiency.
- When multiple robots are available, the plan should prioritize parallel task execution and collaboration whenever possible.
- Tasks that are independent and non-conflicting should be scheduled within the same Execution Step to enable parallel execution.
- Actions assigned to different robots within the same step execute in parallel, while actions listed for a single robot execute sequentially.
- The time cost of an Execution Step equals the maximum number of actions assigned to any robot in that step. In other words, the robot with the longest action sequence determines the duration of that step.
- Therefore, the task plan should aim to:
    - Minimize the total number of Execution Steps
    - Maximize parallel execution across robots
    - Balance workloads among robots when possible
When planning, explicitly attempt to maximize robot utilization and minimize makespan by scheduling as many independent actions as possible in parallel.
## III. Execution Step Organization and Parallel Semantics
The plan is composed of **Execution Steps**, each containing concrete executable actions.
Execution rules:
- If **different robots** have actions within the **same Execution Step**, their **entire action sequences start simultaneously and execute in parallel**;
- If a **single robot** has multiple actions within a step, those actions must be **listed in order and executed sequentially**;
- An Execution Step completes only after all participating robots finish their assigned actions (step-level synchronization).
